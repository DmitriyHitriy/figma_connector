# Требования к модулю авторизации

## Аутентификация
- Система должна предоставлять права входа в клиент при верно введенных логине и пароле, если пользователь имеет права доступа к клиенту
- Система должна предоставлять права входа в клиент при верно введенном адресе сервера, если пользователь имеет права доступа к клиенту
- Система должна обеспечивать авторизацию пользователей с ролью Администратор
- Система должна обеспечивать авторизацию пользователей с ролью Оператор
- Оператор с валидными данными получает немедленный доступ независимо от настройки сцены/устройств
- Система не должна предоставлять доступ Оператору, если клиент не добавлен Администратором

## Поля ввода
- Система должна иметь альтернативные методы ввода данных для авторизации
- Система должна иметь возможность смены языка ввода
- Система должна отображать текущий язык ввода
- Система должна иметь возможность переключения между полями ввода с клавиатуры
- Для поля "Логин" установлены ограничения
- Для поля "Пароль" установлены ограничения
- Для поля "Адрес" установлены ограничения
- Система должна предоставлять пользователю возможность временно просмотреть введенный текст в поле пароля

## Обработка ошибок
- Уведомление об ошибке при неверном логине и/или пароле
- Уведомление пользователя без прав доступа о невозможности подключения
- Блокировка клиента на 15 секунд при 3+ неверных попытках
- При недоступности сервера — заглушка с авто-переподключением и отменой

## Журналирование
- Фиксация всех попыток авторизации в БД
- Отображение попыток в Журнале событий
- Фиксация: успешный вход, неудачный вход, выход, завершение сессии администратором

## Автоматический вход
- Локальное сохранение учётных данных (логин/пароль) для автоматического входа
- При запуске с сохранёнными данными — автоматическая авторизация
- Возможность отменить автоматический вход до его выполнения
- Если снять флажок "Сохранить данные" — удаление сохранённых данных после успешной авторизации

## Роли
- Администратор имеет доступ в Клиент
- Оператор с валидными данными получает немедленный доступ (независимо от настройки сцены/устройств)
- Оператор НЕ получает доступ, если клиент не добавлен Администратором
- При изменении конфигурации Администратором — обновление у Оператора без ручного обновления
- При принудительном завершении сеанса — перенаправление на страницу авторизации
