# Модуль "Объекты охраны" — Требования к реализации

## 1. Термины и определения

1.1. АРМ-А — Автоматизированное рабочее место администратора.
1.2. SECOBJ — Объект охраны.
1.3. Объекты охраны — объекты (сооружения, здания, помещения, площадки с прилегающими территориями), оснащенные специализированными устройствами (датчиками, видеокамерами), служащими источниками информации. Могут быть вложены друг в друга (кабинет на этаже, этаж в здании).
1.4. Схема объекта — планиметрическое отображение объекта охраны.
1.5. Устройство — аппаратное средство для сбора, передачи и обработки данных в системе охраны.
1.6. Инцидент — событие, влекущее нежелательные последствия с отклонением от штатного режима работы объекта, требующее ответной реакции.
1.7. STS-107 — извещатель охранный для обнаружения вторжения нарушителя.
1.8. STS-111 — извещатель охранный с комплектом чувствительных элементов для обнаружения нарушителя, пытающегося повредить/преодолеть заграждение.
1.9. STS-123 — линейный двухпозиционный оптико-электронный извещатель для обнаружения человека, пересекающего рубеж.
1.10. STS-103 — линейный пассивный оптико-электронный извещатель для охраны протяженных участков.

## 2. Предустановленный объект "Внешний мир"

2.1. Система отображает предустановленный объект охраны "Внешний мир" в качестве корневого элемента иерархии.
2.2. "Внешний мир" находится на самом верхнем уровне и не имеет родительского объекта.
2.3. Объект "Внешний мир" доступен сразу после первого входа в систему.
2.4. Запрещено удаление "Внешний мир".
2.5. Запрещено редактирование наименования "Внешний мир".
2.6. Разрешено редактирование схемы объекта "Внешний мир".

## 3. Создание объекта охраны

### 3.1. Иерархия и добавление
3.1.1. Администратор может добавить объект охраны.
3.1.2. При наведении на элемент добавления отображается хинт-подсказка.
3.1.3. При создании объекта отображается поле для ввода наименования.

### 3.2. Валидация наименования
3.2.1. Допустимые символы:
  - Латинские буквы A-Z, a-z
  - Кириллические буквы А-Я, а-я
  - Цифры 0-9
  - Спецсимволы: ! @ # $ % ^ & * ( ) _ - = + [ ] { } ; : ' " , . < > / ? | ~ `
3.2.2. Поле обязательно для заполнения.
3.2.3. Минимальное количество символов: 2.
  - При вводе 1 символа: сообщение о минимальном количестве.
3.2.4. Максимальное количество символов: 25.

### 3.3. Добавление/редактирование/удаление
3.3.1. При добавлении/редактировании/удалении объекта происходит перерисовка иерархии с анимацией.
3.3.2. При добавлении: дочерний объект добавляется в конец списка.
3.3.3. При удалении: объект удаляется из иерархии, список перерисовывается.

## 4. Поиск и фильтрация объектов охраны

4.1. При наличии более 2 объектов отображается строка поиска.
4.2. Поиск осуществляется по наименованиям объектов охраны.
4.3. После ввода значения в строку поиска отображаются только подходящие объекты.
4.4. При удалении значения из строки поиска восстанавливается полный список.

## 5. Отображение списка объектов охраны

### 5.1. Таблица/список
5.1.1. Объекты отображаются списком с основной информацией.
5.1.2. Список содержит колонку наименования.
5.1.3. Иконка схемы объекта (заполненная, если схема загружена; пустая, если нет).
5.1.4. Иконка разворачивания/сворачивания дочерних объектов.
5.1.5. При первом разворачивании родительского объекта его дочерние объекты загружаются асинхронно с анимацией загрузки.

### 5.2. Навигационная цепочка (breadcrumbs)
5.2.1. При выборе объекта в навигационной цепочке отображается полный путь от "Внешний мир" до выбранного объекта.
5.2.2. Пример: "Внешний мир > Объект 1 > Объект 2".
5.2.3. Клик на элемент breadcrumbs открывает соответствующий объект.
5.2.4. Навигационная цепочка отображается также в заголовке карточки объекта.

### 5.3. Карточка объекта
5.3.1. При выборе объекта отображается карточка с подробной информацией.
5.3.2. Карточка содержит: наименование, иерархический путь, статус, количество устройств и инцидентов.

## 6. Работа со схемой объекта

6.1. При нажатии на строку объекта или иконку схемы открывается редактор схемы.
6.2. Зона редактирования схемы имеет размер 300x300 метров (или пропорционально).
6.3. Если размер объекта превышает 300x300 метров или 35% экрана, отображается масштабирование с полосами прокрутки.
6.4. При загрузке элементов схемы отображается анимация загрузки до полной прогрузки фонового изображения, устройств, выделенных областей и текста.

## 7. Удаление объекта охраны

### 7.1. Через список
7.1.1. При наведении на строку объекта появляется иконка контекстного меню.
7.1.2. Контекстное меню содержит кнопку "Удалить объект".
7.1.3. При нажатии открывается окно "Удаление" с подтверждением.

### 7.2. Через карточку
7.2.1. В карточке объекта присутствует иконка контекстного меню.
7.2.2. Контекстное меню содержит кнопку удаления.
7.2.3. При нажатии — окно "Удаление".

### 7.3. Ограничения
7.3.1. Объект "Внешний мир" не может быть удален.
7.3.2. Если у объекта есть дочерние объекты, удаление невозможно без предварительного удаления дочерних.

### 7.4. Окно "Удаление"
7.4.1. Содержит: "Вы хотите удалить объект "Наименование"?" и кнопки отмены, удаления, закрытия.

## 8. Устройства и их отображение

8.1. При нажатии на ячейку с устройствами открывается окно "Устройства".
8.2. В модальном окне отображаются привязанные устройства.
8.3. Каждое устройство отображается в виде карточки.

### 8.1. Характеристики устройств
8.1.1. Для устройств БА-150 отображается: наименование, тип, ID, статус, местоположение.
8.1.2. Для устройств STS-411, STS-409 отображается: наименование, тип, ID, ворота/калитка/проходная.
8.1.3. Устройства STS-411, STS-409 всегда находятся попарно (по 6 шт.).

## 9. Поиск устройств в окне устройств

9.1. Минимальная длина поиска: 2 символа.
9.2. Максимальная длина: 25 символов.
9.3. Допустимые символы: буквы (латиница, кириллица), цифры 0-9, спецсимволы.
9.4. При вводе 1 символа: сообщение "Минимальное количество символов — 2".
9.5. Поиск происходит среди названий устройств.

## 10. Доступ к модулю по ролям

10.1. Просмотр объектов охраны доступен: Администратор, Оператор.
10.2. Создание/редактирование/удаление объектов охраны доступно: Администратор.
10.3. Редактирование схемы объекта доступно: Администратор.
10.4. Просмотр и редактирование устройств доступно: Администратор, Оператор.

## 11. Права доступа (разграничение)

11.1. Система предоставляет возможность разграничения прав доступа для различных ролей:
  - Администратор: полный доступ (создание, редактирование, удаление объектов и устройств, редактирование схем).
  - Оператор: просмотр объектов, просмотр и работа с устройствами (без редактирования схем).

## 12. Инциденты

12.1. При нажатии на ячейку с инцидентами открывается окно "Инциденты".
12.2. В окне отображаются инциденты, связанные с выбранным объектом охраны.

## 13. Валидация полей (для всех модулей ОО)

13.1. Допустимые символы для наименования: буквы (латиница/кириллица), цифры 0-9, спецсимволы ! @ # $ % ^ & * ( ) _ - = + [ ] { } ; : ' " , . < > / ? | ~ `.
13.2. Поля с типом "Обязательное" не могут быть пустыми.
13.3. Минимальная длина текстовых полей: 2 символа.
13.4. Максимальная длина текстовых полей: 25 символов.
13.5. При невалидных значениях: красная подсветка поля + сообщение об ошибке.
