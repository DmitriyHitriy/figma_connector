# Модуль "Пользователи" — Требования к реализации

## 1. Навигация и доступ к вкладке "Пользователи"

1.1. В навигационном меню интерфейса присутствует кнопка для перехода во вкладку "Пользователи".
1.2. Название кнопки: логичное и понятное для оператора (например, "Пользователи").
1.3. При нажатии на кнопку оператор перенаправляется во вкладку "Пользователи".
1.4. Вкладка открывается без ошибок, отображая список всех пользователей.
1.5. Переход во вкладку доступен только авторизованным операторам с правами на управление пользователями.

## 2. Отображение списка пользователей (таблица)

2.1. Пользователи отображаются в виде таблицы с шапкой.
2.2. Колонки таблицы:
  - Фото пользователя (или инициалы, если фото нет)
  - ФИО пользователя
  - Логин пользователя
  - Роль пользователя
  - Статус пользователя ("Текущий пользователь" / "Активен" / "Заблокирован")
2.3. Текущий авторизованный пользователь имеет статус "Текущий пользователь".

## 3. Формирование инициалов для аватара

3.1. Разделение значения по пробелу и дефису; для инициалов используется первое значимое слово.
3.2. Инициалы = первая буква имени + первая буква фамилии (всегда 2 символа).
3.3. Отчество игнорируется при наличии фамилии.
3.4. При отсутствии фамилии инициалы = первая буква имени + первая буква отчества.
3.5. Примеры:
  - "Андрей Генрихович Валентинов-Апполонов" → "АВ"
  - "Анна-Мария Валентиновна Григорьева-Апполонова" → "АГ"
  - "Анна-Мария Валентиновна" → "АВ"
3.6. Инициалы не смещаются и не обрезаются в превью аватарки.

## 4. Добавление пользователя

### 4.1. Учетная запись администратора при первом развертывании
  - Роль: Администратор
  - Логин: admin
  - Пароль: Adm123

### 4.2. Кнопка добавления пользователя
  - Кнопка всегда доступна и находится в видимой области экрана.
  - Подсказка "Нажмите на плюс для добавления пользователя" отображается после прокрутки списка до конца.

### 4.3. Окно "Добавление пользователя"
  - Открывается при нажатии на кнопку "Добавить пользователя".
  - Содержит:
    - Кнопку добавления фото
    - Обязательные поля: Фамилия, Имя, Логин, Пароль, Подтверждение пароля, выпадающий список Роль
    - Необязательное поле: Отчество
  - Закрытие окна только при нажатии на: пиктограмму "Закрыть", кнопку "Отмена", кнопку "Добавить", клавишу Esc.
  - Нажатие вне области окна не закрывает его.

### 4.4. Добавление фото пользователя
  - Форматы: PNG, JPG.
  - Максимальный размер: 30 МБ.
  - После добавления фото автоматически масштабируется до 400x400 px.

### 4.5. Редактирование и удаление фото
  - После добавления фото доступны редактирование и удаление.
  - Редактирование: открывается окно "Редактирование фото" с Crop Tool, кнопками отмены, сохранения и закрытия. После CropTool и сохранения обрезанная область масштабируется до 400x400 px.
  - Удаление: окно "Удаление" с текстом "Вы хотите удалить фото?" и кнопками отмены, удаления и закрытия.

### 4.6. Валидация полей (Фамилия, Имя, Отчество)
  - Макс. длина: 25 символов.
  - Мин. длина: 2 символа.
  - Допустимые символы: латиница a-z/A-Z, кириллица а-я/А-Я, дефис (-) только внутри строки, апостроф (') только внутри строки, пробелы между частями слов (например, "де ла Круз").
  - Запрещено: начало со спецсимволов (-, '), более одного спецсимвола подряд, начало с пробела, более одного пробела подряд.
  - Поведение:
    - 0 символов: красная подсветка + "Обязательное поле", кнопка "Добавить" неактивна.
    - 1 символ: красная подсветка + "Минимальное количество символов — 2", кнопка неактивна.
    - >= 2 символов: ошибки скрыты, кнопка активна.

### 4.7. Валидация поля Логин
  - Макс. длина: 25 символов.
  - Мин. длина: 3 символа.
  - Допустимые символы: буквы (латиница, кириллица), цифры 0-9, подчёркивание (_), точка (.), дефис (-).
  - Чувствителен к регистру.
  - Запрещено: начинаться/заканчиваться спецсимволами (-, _, .), два спецсимвола подряд.
  - Поведение:
    - 0 символов: "Обязательное поле", кнопка неактивна.
    - 1-2 символа: "Минимальное количество символов — 3", кнопка неактивна.
    - >= 3 символов: ошибки скрыты, кнопка активна.

### 4.8. Валидация выпадающего списка Роль
  - Не выбрано: красная подсветка + "Обязательное поле", кнопка неактивна.
  - Выбрано: ошибка скрыта, кнопка активна.

### 4.9. Валидация полей Пароль и Подтверждение пароля
  - Пиктограмма глаза для показа/скрытия пароля.
  - Допустимые символы: латиница A-Z/a-z, цифры 0-9, спецсимволы ! @ # $ % ^ & * ( ) _ - = + [ ] { } ; : ' " , . < > / ? | ~ `.
  - Запрещено: пробелы, недопустимые символы (эмодзи и пр.).
  - Требования: минимум 1 заглавная, 1 строчная, 1 цифра; не более 2 одинаковых символов подряд.
  - Мин. длина: 6 символов. Макс. длина: 25 символов.
  - Ошибки поля "Пароль":
    1. "Поле "Пароль" является обязательным" (0 символов)
    2. "Пароль содержит пробел..."
    3. "Пароль содержит недопустимые символы..."
    4. "Минимальное количество символов — 6" (< 6)
    5. "Пароль должен содержать минимум одну заглавную букву, одну строчную букву и одну цифру"
    6. "Пароль не может содержать более двух одинаковых символов подряд"
  - Ошибки поля "Подтверждение пароля":
    1. "Поле "Подтверждение пароля" является обязательным" (0 символов)
    2. "Введенные пароли не совпадают"
  - Ошибки отображаются только в пределах соответствующего поля, без дублирования.

### 4.10. Обновление после добавления
  - После нажатия "Добавить" пользователь сразу появляется в списке.
  - Счетчики в сайд-меню обновляются.

## 5. Редактирование пользователя

### 5.1. Редактирование в сайд-меню
  - При нажатии на строку пользователя в сайд-меню отображается информация с возможностью редактирования.
  - Редактируемые данные:
    - Фото пользователя (пиктограммы при наведении: удаление, редактирование; при отсутствии фото — пиктограмма добавления)
    - Фамилия (поле ввода, обязательное)
    - Имя (поле ввода, обязательное)
    - Отчество (поле ввода, необязательное)
    - Пароль (кнопка перехода в окно "Изменение пароля")
  - Валидация полей Фамилия, Имя, Отчество: см. п. 4.6.
  - Кнопки "Отмена" и "Сохранить" (до внесения изменений неактивны).

### 5.2. Подтверждение изменений
  - При нажатии "Сохранить" → окно "Список изменений" с отображением внесенных изменений и кнопками: отмена, подтверждение, закрытие.
  - При нажатии "Отмена" → окно "Отмена редактирования" с текстом "Вы хотите сбросить внесенные изменения?" и кнопками: отмена, сброс, закрытие.

### 5.3. Обновление после редактирования
  - После сохранения изменения отображаются в списке пользователей и в сайд-меню.

### 5.4. Типы учетных записей, доступных для редактирования
  - Встроенный Администратор
  - Администраторы, добавленные вручную
  - Операторы, добавленные вручную

## 6. Изменение роли пользователя

6.1. В сайд-меню отображается выпадающий список с текущей ролью пользователя.
6.2. При раскрытии списка можно выбрать только одну роль.
6.3. После выбора новой роли — кнопки "Отмена" и "Сохранить".
6.4. При сохранении — окно "Список изменений", при отмене — окно "Отмена редактирования".
6.5. После сохранения роль обновляется в списке пользователей и в сайд-меню.

## 7. Удаление пользователя

### 7.1. Через список пользователей
  - ПКМ на строке пользователя → контекстное меню → кнопка "Удалить" → окно "Удаление".

### 7.2. Через сайд-меню
  - Пиктограмма открытия контекстного меню → кнопка "Удалить" → окно "Удаление".

### 7.3. Ограничения
  - Текущий авторизованный администратор не может быть удален:
    - В списке: ПКМ не вызывает контекстное меню.
    - В сайд-меню: меню с действиями не отображается.
    - Чекбокс группового выбора не отображается.

### 7.4. Окно "Удаление"
  - Текст: "Вы хотите удалить пользователя "ФИО"?"
  - Кнопки: отмена, удаление, закрытие.

### 7.5. Обновление после удаления
  - Список пользователей обновляется.
  - Счетчики в сайд-меню обновляются.

## 8. Поиск и фильтрация пользователей

### 8.1. Поле поиска
  - Расположение: над списком пользователей.
  - Состав: поле ввода + пиктограмма поиска.
  - Хинт: "Поиск по ФИО или логину пользователя".
  - Макс. длина: 50 символов. Мин. длина для поиска: 2 символа.

### 8.2. Логика поиска
  - Автоматический поиск без дополнительных кнопок.
  - Поиск по полям: ФИО, логин (одновременно).
  - Поддержка произвольного порядка ввода.
  - Частичные совпадения.
  - Корректная обработка пробелов и разделителей.
  - Примеры:
    - Ввод "admin Сергеевич Иванов Петр" → находит пользователя с логином "admin", фамилией "Иванов", именем "Петр", отчеством "Сергеевич".
    - Ввод "Иванов Сергеевич" → все пользователи с фамилией "Иванов" и отчеством "Сергеевич".
    - Ввод "Петр admin" → пользователь с именем "Петр" и логином "admin".

### 8.3. Фильтр
  - Расположение: над списком пользователей.
  - Состав: выпадающий список ролей (мультивыбор) + выпадающий список статусов (один выбор: активные / заблокированные).
  - Индикатор выбранного фильтра рядом с кнопкой "Фильтры".
  - Кнопка сброса всех фильтров.

## 9. Блокировка / Разблокировка пользователя

### 9.1. Через список пользователей
  - ПКМ на строке → контекстное меню → кнопка блокировки/разблокировки → окно "Блокировка" или "Разблокировка".

### 9.2. Через сайд-меню
  - Пиктограмма контекстного меню → кнопка блокировки/разблокировки → соответствующее окно.

### 9.3. Ограничения
  - Текущий авторизованный администратор: недоступна блокировка/разблокировка, контекстное меню не отображается, чекбокс группового выбора не отображается.

### 9.4. Окна
  - "Блокировка": "Вы хотите заблокировать пользователя "ФИО"?" + кнопки отмены, блокировки, закрытия.
  - "Разблокировка": "Вы хотите разблокировать пользователя "ФИО"?" + кнопки отмены, разблокировки, закрытия.

### 9.5. Обновление
  - Статус обновляется в списке и в сайд-меню.

## 10. Статистика в сайд-меню (общая)

### 10.1. Общая статистика
  - Заголовок "Пользователи"
  - Заголовок "Общая информация":
    - "Всего пользователей" (количество)
    - "Заблокировано" (количество)
    - "Активно" (количество)
  - Заголовок "Роли доступа":
    - "Администратор" (количество)
    - "Пользователь" (количество)

### 10.2. Условия отображения
  - Статистика отображается всегда, кроме случаев:
    - Нажат пользователь в списке (сайд-меню показывает редактирование).
    - Нажат чекбокс группового выбора.
  - При клике в пустую область экрана — возврат к статистике.

### 10.3. Автообновление счетчиков при любых изменениях.

## 11. Статистика в сайд-меню (по выбранным пользователям)

### 11.1. Статистика по выбранным
  - Заголовок "Пользователи"
  - Заголовок "Общая информация":
    - "Выбрано пользователей" (количество)
    - "Выбрано заблокированных" (количество)
    - "Выбрано активных" (количество)
  - Заголовок "Роли доступа":
    - "Выбрано Администраторов" (количество)
    - "Выбрано Пользователей" (количество)

### 11.2. Условия отображения
  - Только при нажатии на чекбокс группового выбора.
  - Нельзя выбрать текущего авторизованного пользователя.

## 12. Изменение пароля пользователя

### 12.1. Кнопка изменения пароля в сайд-меню
  - При нажатии открывается окно "Изменение пароля".

### 12.2. Окно "Изменение пароля"
  - Поля ввода пароля защищены (символы скрыты).
  - Содержит: поле пароля, поле подтверждения пароля, кнопки отмены, сохранения, закрытия.

### 12.3. Валидация
  - Идентично п. 4.9 (пиктограмма глаза, ограничения, ошибки).

### 12.4. Применение
  - После сохранения пользователь может войти только с новым паролем.

## 13. Групповое взаимодействие

### 13.1. Чекбоксы
  - При наведении на строку отображается чекбокс.
  - При нажатии на чекбокс в сайд-меню — количество выбранных пользователей.

### 13.2. Поведение при выборе
  - После выделения первого пользователя через чекбокс, последующие выделяются нажатием на любой элемент строки.
  - Повторное нажатие на выделенного пользователя снимает выделение.

### 13.3. Меню группового взаимодействия
  - Появляется в блоке поиска и фильтрации при нажатии на чекбокс.
  - Содержит:
    - Кнопка "Заблокировать выбранных"
    - Кнопка "Разблокировать выбранных"
    - Кнопка "Удалить выбранных"
    - Чекбокс "Выбрать все"

### 13.4. Сброс группового выбора
  - Только при нажатии на чекбокс "Выбрать все" или клавишу Esc.

### 13.5. Групповое удаление
  - Окно "Удаление": "Вы хотите удалить выбранных пользователей (N) ?"
  - Текущий авторизованный администратор не может быть выбран/удален.
  - "Выбрать все" не выбирает текущего авторизованного администратора.
  - После удаления: список обновляется, счетчики обновляются.

### 13.6. Групповая блокировка/разблокировка
  - Окно "Блокировка": "Вы хотите заблокировать выбранных пользователей (N) ?"
  - Окно "Разблокировка": "Вы хотите разблокировать выбранных пользователей (N) ?"
  - Текущий авторизованный администратор не выбирается.
  - Если все выбранные "Активен" → кнопка "Разблокировать" неактивна.
  - Если все выбранные "Заблокирован" → кнопка "Заблокировать" неактивна.
  - Если и активные, и заблокированные → все кнопки активны.
  - После операции: статусы обновляются.

## 14. База данных — пользователи

14.1. Хранение информации о пользователях в БД: фото, логин, фамилия, имя, отчество, роль.
14.2. Администратор имеет доступ ко всей сохраненной информации через интерфейс.

## 15. База данных — пароли

15.1. Пароли хранятся в БД в хешированном виде.
15.2. При добавлении пользователя: пароль хешируется перед сохранением.
15.3. При изменении пароля: старый хеш заменяется новым, новый пароль хешируется перед сохранением.

## 16. База данных — архивирование логинов удаленных пользователей

16.1. Логины удаленных пользователей архивируются в БД.
16.2. История событий удаленных пользователей остается доступной в журнале событий.
16.3. В списке пользователей отображаются все, кроме удаленных.
16.4. Запрещено повторное добавление пользователя с архивным логином.
16.5. При попытке: сообщение "Указанный логин принадлежит архивированному пользователю. Укажите другой логин."
